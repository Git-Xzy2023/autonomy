# Vue2 异步更新队列源码

## 为什么需要异步更新

数据变化时，如果立即触发 Watcher 重新渲染，会有两个问题：

1. **性能浪费**：循环里 `this.count++` 100 次，会触发 100 次 render，但最终只需要 1 次。
2. **一致性**：同步更新可能导致"中间状态"被渲染。

Vue 的策略：**数据变化时不立即更新，把 Watcher 入队，在 nextTick 统一 flush**。这样多次数据变化只触发一次 render。

## 源码解析

### 1. Watcher 入队

```js
// src/core/observer/scheduler.js
const queue = [];       // 待执行的 Watcher 队列
let has = {};           // 去重表（id → true）
let waiting = false;    // 是否已等待 nextTick
let flushing = false;   // 是否正在 flush
let index = 0;          // flush 时的当前索引

export function queueWatcher(watcher) {
  const id = watcher.id;
  if (has[id] == null) {
    has[id] = true;
    if (!flushing) {
      queue.push(watcher); // 入队
    } else {
      // flush 过程中新加入的 Watcher，按 id 插入合适位置
      let i = queue.length - 1;
      while (i > index && queue[i].id > watcher.id) {
        i--;
      }
      queue.splice(i + 1, 0, watcher);
    }
    if (!waiting) {
      waiting = true;
      nextTick(flushSchedulerQueue); // 在 nextTick 执行 flush
    }
  }
}
```

### 2. flush 队列

```js
function flushSchedulerQueue() {
  flushing = true;
  // 按 id 排序：父组件 Watcher 先于子组件（父先创建）
  // user Watcher 先于 render Watcher（先执行用户的 watch 回调）
  // 如果组件父 Watcher 在 flush 中被销毁，跳过其子 Watcher
  queue.sort((a, b) => a.id - b.id);

  for (index = 0; index < queue.length; index++) {
    const watcher = queue[index];
    has[watcher.id] = null;
    watcher.run(); // 重新求值 + 触发回调
  }

  // 重置
  index = queue.length = 0;
  has = {};
  waiting = flushing = false;
}
```

### 3. nextTick 实现

```js
// src/core/util/next-tick.js
const callbacks = [];
let pending = false;

export function nextTick(cb, ctx) {
  let _resolve;
  callbacks.push(() => {
    if (cb) {
      cb.call(ctx);
    } else if (_resolve) {
      _resolve(ctx);
    }
  });
  if (!pending) {
    pending = true;
    timerFunc(); // 异步触发 flushCallbacks
  }
  if (!cb && typeof Promise !== 'undefined') {
    return new Promise(resolve => { _resolve = resolve; });
  }
}

function flushCallbacks() {
  pending = false;
  const copies = callbacks.slice(0);
  callbacks.length = 0;
  for (let i = 0; i < copies.length; i++) {
    copies[i]();
  }
}

// 微任务降级策略
let timerFunc;
if (typeof Promise !== 'undefined' && isNative(Promise)) {
  // 1. Promise.then（微任务）
  const p = Promise.resolve();
  timerFunc = () => {
    p.then(flushCallbacks);
    if (isIOS) setTimeout(noop); // iOS bug 修复
  };
} else if (!isIE && typeof MutationObserver !== 'undefined') {
  // 2. MutationObserver（微任务）
  let counter = 1;
  const observer = new MutationObserver(flushCallbacks);
  const textNode = document.createTextNode(String(counter));
  observer.observe(textNode, { characterData: true });
  timerFunc = () => {
    counter = (counter + 1) % 2;
    textNode.data = String(counter);
  };
} else if (typeof setImmediate !== 'undefined') {
  // 3. setImmediate（宏任务，IE 专用）
  timerFunc = () => { setImmediate(flushCallbacks); };
} else {
  // 4. setTimeout（宏任务，兜底）
  timerFunc = () => { setTimeout(flushCallbacks, 0); };
}
```

## 关键设计点

**1. 去重（has 表）**

同一个 Watcher 在一次事件循环里只入队一次。即使数据被改了 100 次，只 render 一次。

**2. 排序（id 升序）**

Watcher 创建顺序：父组件 → 子组件 → user watcher。按 id 排序保证：

- 父组件先更新（避免子组件用旧 props 渲染后又被父更新覆盖）
- user watcher 先于 render watcher（用户回调先执行，再渲染）

**3. 微任务优先**

Vue2 优先用 `Promise.then`（微任务），保证 DOM 更新在当前事件循环结束前完成，用户感知不到延迟。降级链：Promise → MutationObserver → setImmediate → setTimeout。

**4. flush 过程中新加入的 Watcher**

如果 flush 时又触发了新的 Watcher（比如 watch 回调里改了数据），新 Watcher 按 id 插入队列，保证顺序正确。

## nextTick 的本质

`nextTick` 就是"把回调塞进微任务队列，在 DOM 更新后执行"。所以：

```js
this.msg = 'hello';
this.$nextTick(() => {
  // 这里能拿到更新后的 DOM
  console.log(this.$el.textContent); // 'hello'
});
```

`$nextTick` 和数据更新用的是同一个 `callbacks` 队列，且 `flushSchedulerQueue` 先入队（数据变化时先 `nextTick(flushSchedulerQueue)`），用户调 `$nextTick` 后入队，所以用户回调在 DOM 更新后执行。
