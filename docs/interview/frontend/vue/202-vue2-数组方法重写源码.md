# Vue2 数组方法重写源码

## 为什么数组要特殊处理

`Object.defineProperty` 监听数组有两个问题：

1. **性能问题**：数组可能有成千上万个元素，逐个 `defineReactive` 开销大。
2. **无法监听索引赋值**：`arr[0] = x` 不触发 setter（除非提前 `defineReactive(arr, 0)`）。
3. **无法监听 length 修改**：`arr.length = 0` 不触发。

Vue2 的解决方案：**重写数组的 7 个变异方法**（会修改原数组的方法），在方法内部手动触发通知。

## 源码解析

```js
// src/core/observer/array.js

const arrayProto = Array.prototype;
// 创建一个继承自 Array.prototype 的对象，避免污染原生原型
export const arrayMethods = Object.create(arrayProto);

const methodsToPatch = [
  'push', 'pop', 'shift', 'unshift', 'splice', 'sort', 'reverse'
];

methodsToPatch.forEach(function (method) {
  // 缓存原方法
  const original = arrayProto[method];
  // 在 arrayMethods 上重写
  def(arrayMethods, method, function mutator(...args) {
    const result = original.apply(this, args); // 先执行原方法
    const ob = this.__ob__; // 拿到 Observer 实例
    let inserted;
    switch (method) {
      case 'push':
      case 'unshift':
        inserted = args; // push/unshift 新增的元素
        break;
      case 'splice':
        inserted = args.slice(2); // splice(index, deleteCount, ...items) 的 items
        break;
    }
    if (inserted) {
      ob.observeArray(inserted); // 新增元素也要响应式化
    }
    // 触发更新（用 __ob__.dep，不是属性的 dep）
    ob.dep.notify();
    return result;
  });
});
```

## 如何让数组用上重写的方法

```js
// src/core/observer/index.js
class Observer {
  constructor(value) {
    this.value = value;
    this.dep = new Dep();
    def(value, '__ob__', this);

    if (Array.isArray(value)) {
      if (hasProto) {
        // 支持__proto__：直接替换原型
        protoAugment(value, arrayMethods);
      } else {
        // 不支持__proto__（IE10-）：逐个拷贝方法到数组上
        copyAugment(value, arrayMethods, arrayKeys);
      }
      this.observeArray(value); // 递归观察数组元素
    } else {
      this.walk(value);
    }
  }
}

function protoAugment(target, src) {
  target.__proto__ = src; // value.__proto__ = arrayMethods
}

function copyAugment(target, src, keys) {
  for (let i = 0; i < keys.length; i++) {
    const key = keys[i];
    def(target, key, src[key]); // 在数组自身定义方法
  }
}
```

## 数组依赖收集的特殊处理

数组本身没有 `defineReactive`，那它的依赖怎么收集？答案在 `defineReactive` 的 getter 里：

```js
get: function reactiveGetter() {
  const value = val;
  if (Dep.target) {
    dep.depend();
    if (childOb) {
      childOb.dep.depend(); // 子对象（数组）的 __ob__.dep 也收集依赖
      if (Array.isArray(value)) {
        dependArray(value); // 递归让数组里的对象元素也收集依赖
      }
    }
  }
  return value;
}

function dependArray(value) {
  for (let i = 0; i < value.length; i++) {
    const e = value[i];
    if (e && e.__ob__) {
      e.__ob__.dep.depend();
    }
    if (Array.isArray(e)) {
      dependArray(e); // 嵌套数组递归
    }
  }
}
```

所以数组变化时，调用 `ob.dep.notify()` 就能通知到所有访问过该数组的 Watcher。

## 无法覆盖的场景

```js
arr[0] = newValue;       // ❌ 不触发更新
arr.length = 0;          // ❌ 不触发更新
arr[5] = 1;              // ❌ 不触发更新
```

**解决方案：**

```js
// 1. 用 $set
this.$set(this.arr, 0, newValue);

// 2. 用 splice
this.arr.splice(0, 1, newValue);

// 3. 清空用 splice
this.arr.splice(0, this.arr.length);
```

## Vue3 的改进

Vue3 用 Proxy 拦截整个对象，`arr[0] = x` 和 `arr.length = 0` 都能被 `set` trap 捕获，不再需要重写数组方法。这是 Vue3 响应式更完整的根本原因。
