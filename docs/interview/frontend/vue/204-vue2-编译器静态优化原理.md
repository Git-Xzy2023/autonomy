# Vue2 编译器静态优化原理

## 为什么需要静态优化

Vue2 的 patch 是**全量 diff**：新旧 VNode 树逐层比较，即使大部分节点是静态的（没有动态绑定），每次更新也要遍历它们做对比。如果能标记出静态节点，diff 时跳过它们，能显著提升更新性能。

## 三层优化

### 1. 静态节点标记（optimize）

编译时遍历 AST，标记 `static` 和 `staticRoot`：

```js
// src/compiler/optimizer.js
export function optimize(root, options) {
  if (!root) return;
  markStatic(root);       // 标记静态节点
  markStaticRoots(root);  // 标记静态根节点
}

function markStatic(node) {
  node.static = isStatic(node);
  if (node.type === 1) { // 元素节点
    for (let i = 0; i < node.children.length; i++) {
      const child = node.children[i];
      markStatic(child);
      if (!child.static) {
        node.static = false; // 子节点非静态，父节点也非静态
      }
    }
  }
}

function isStatic(node) {
  if (node.type === 2) { // 表达式 {{ }}
    return false;
  }
  if (node.type === 3) { // 纯文本
    return true;
  }
  // 元素节点：没有 v-if/v-for/v-model 等动态指令，且没有动态绑定
  return !!(node.pre || (
    !node.hasBindings &&
    !node.if && !node.for &&
    !isBuiltInTag(node.tag) &&
    !isDirectChildOfTemplateFor(node) &&
    Object.keys(node).every(key => !isReservedTag(key))
  ));
}

function markStaticRoots(node) {
  if (node.type === 1) {
    if (node.static || node.once) {
      node.staticRoot = true;
      return; // 静态根节点不再深入标记子节点
    }
    // 有单个静态子节点且非纯文本，标记为 staticRoot
    if (node.children.length === 1 && node.children[0].type === 1) {
      node.staticRoot = true;
    }
  }
}
```

### 2. 静态节点提升到 render 函数外

编译时，静态节点被提取成常量，避免每次 render 重新创建 VNode：

```js
// 编译前
<div>
  <h1>标题</h1>
  <p>{{ msg }}</p>
</div>

// 编译后（简化）
const _hoisted_1 = _c('h1', [_v('标题')]); // 静态节点提到外面

function render() {
  return _c('div', [
    _hoisted_1, // 直接引用，不重新创建
    _c('p', [_v(_s(msg))])
  ]);
}
```

### 3. staticRenderFns

静态根节点（`staticRoot`）被抽到独立的 `staticRenderFns` 数组，render 时合并：

```js
// 编译输出
{
  render: function() {
    return _c('div', [
      _m(0), // 调用 staticRenderFns[0]
      _c('p', [_v(_s(msg))])
    ]);
  },
  staticRenderFns: [
    function() {
      return _c('header', [
        _c('h1', [_v('标题')]),
        _c('nav', [_c('a', [_v('链接')])])
      ]);
    }
  ]
}
```

`_m(index)` 是 `renderStatic`，缓存执行结果，后续直接返回缓存。

## 局限性

**1. 静态判断粗糙**

`isStatic` 只能判断"没有动态指令"，无法判断"动态绑定的值是否实际变化"：

```html
<!-- :class="staticClass" 即使 staticClass 永远不变，也不是 static 节点 -->
<div :class="staticClass">...</div>
```

**2. 无法跳过 diff**

即使标记了 `static`，patch 时仍会遍历静态节点做对比（只是跳过 patchVnode 内部的更新）。无法像 Vue3 那样"完全跳过"。

**3. 组件无法静态化**

组件只要有 props 就不是静态的，即使 props 永远不变。

## Vue3 的改进

Vue3 的编译优化是质的飞跃：

1. **PatchFlag**：每个动态节点标记"哪些部分是动态的"（class/style/text/props/...），diff 时只比较标记的部分。
2. **Block Tree**：把动态节点收集到根节点的 `dynamicChildren` 数组，diff 时只遍历动态节点，跳过所有静态节点。
3. **静态提升**：静态节点和静态属性都提升到 render 函数外，且能被 Tree Shaking。
4. **缓存事件处理函数**：`@click="handler"` 编译成缓存函数，避免每次 render 创建新函数导致子组件不必要的更新。

```js
// Vue3 编译输出
const _hoisted_1 = createElementVNode("h1", null, "标题"); // 静态提升

export function render(_ctx, _cache) {
  return (openBlock(), createElementBlock("div", null, [
    _hoisted_1,
    createElementVNode("p", null, toDisplayString(_ctx.msg), 1 /* TEXT */)
    //                                                              ↑ PatchFlag
  ]));
}
```
