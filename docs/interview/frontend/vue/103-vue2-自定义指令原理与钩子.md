# Vue2 自定义指令原理与钩子

## 为什么会有这个功能

有些 DOM 操作无法通过"数据驱动"表达，比如：自动聚焦 input、拖拽、权限按钮（无权限移除元素）、水印、长按。直接操作 DOM 违背 Vue 的声明式理念，自定义指令是 Vue 提供的"受控逃生舱"，把 DOM 操作封装成可复用的指令。

## 用法

```js
// 全局注册
Vue.directive('focus', {
  inserted(el) {
    el.focus();
  }
});

// 局部注册
export default {
  directives: {
    focus: { inserted(el) { el.focus(); } }
  }
}

// 使用
<input v-focus />
```

## 钩子函数

| 钩子         | 触发时机                                  |
| ------------ | ----------------------------------------- |
| bind         | 指令第一次绑定到元素（只调用一次）        |
| inserted     | 绑定元素插入到父节点                      |
| update       | 所在组件 VNode 更新，**可能发生在子组件更新之前** |
| componentUpdated | 所在组件及其子组件全部更新后              |
| unbind       | 指令与元素解绑（只调用一次）              |

每个钩子参数：`(el, binding, vnode, oldVnode)`

- `binding`：`{ name, value, oldValue, expression, arg, modifiers }`
- `v-model="msg"` → `name: 'model'`, `value: msg 的值`, `expression: 'msg'`
- `v-mydirective:foo.bar="msg"` → `arg: 'foo'`, `modifiers: { bar: true }`

## 原理

**1. 编译阶段**

模板编译时，`v-xxx` 被识别为指令，生成到 VNode 的 `directives` 数组：

```js
// <input v-focus /> 编译后
_c('input', { directives: [{ name: 'focus', rawName: 'v-focus' }] })
```

**2. 运行时**

patch 阶段调用 `updateDirectives`，对比新旧指令，调用对应的钩子：

```js
// 源码 src/core/view/directives.js
function updateDirectives(oldVnode, vnode) {
  if (oldVnode.data.directives || vnode.data.directives) {
    _update(oldVnode, vnode);
  }
}

  // ...
}
```

**3. 指令合并**

同一个元素上的多个指令会按定义顺序执行，`customDirectives` 会和内置指令（`v-model`、`v-show`）合并。

## 弊端与 Vue3 的改进

**Vue2 的问题：**

1. **钩子名称与组件生命周期不一致**（bind/inserted vs created/mounted），心智负担重。
2. **`update` 和 `componentUpdated` 容易混淆**，实际开发很少用 `update`（因为子组件可能还没更新）。
3. **指令无法访问组件实例的响应式数据**，只能通过 `binding.expression` 字符串 + `vnode.context` 间接取值。

**Vue3 的改进：**

- 钩子重命名为 `created / beforeMount / mounted / beforeUpdate / updated / beforeUnmount / unmounted`，与组件生命周期对齐。
- 移除 `bind` / `inserted` / `update` / `componentUpdated`。
- 指令能直接拿到组件实例（通过 `binding.instance`）。
