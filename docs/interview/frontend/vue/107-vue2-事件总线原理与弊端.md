# Vue2 事件总线原理与弊端

## 为什么会有这个功能

Vue 组件通信方式有：`props/$emit`（父子）、`provide/inject`（祖先-后代）、`$refs`（父访问子）、Vuex（全局状态）。但有些场景是**兄弟组件**或**跨层级组件**之间需要通信，又不想引入 Vuex。`EventBus`（事件总线）利用 Vue 实例的 `$on/$emit/$off` API，实现一个全局发布订阅。

## 用法

```js
// 创建总线
const bus = new Vue();

// 全局注入
Vue.prototype.$bus = bus;

// 组件 A 监听
export default {
  mounted() {
    this.$bus.$on('refresh', (data) => {
      console.log('收到', data);
    });
  },
  beforeDestroy() {
    this.$bus.$off('refresh'); // 必须手动解绑！
  }
}

// 组件 B 触发
this.$bus.$emit('refresh', { id: 1 });
```

## 原理

`EventBus` 本质就是一个 Vue 实例，复用 Vue 的事件系统：

```js
// 源码 src/core/instance/events.js
Vue.prototype.$on = function(event, fn) {
  const vm = this;
  if (Array.isArray(event)) {
    event.forEach(e => vm.$on(e, fn));
  } else {
    (vm._events[event] || (vm._events[event] = [])).push(fn);
  }
  return vm;
};

Vue.prototype.$emit = function(event, ...args) {
  const vm = this;
  const cbs = vm._events[event];
  if (cbs) {
    cbs.forEach(fn => fn.apply(vm, args));
  }
  return vm;
};

Vue.prototype.$off = function(event, fn) {
  const vm = this;
  if (!arguments.length) {
    vm._events = Object.create(null);
    return vm;
  }
  // 移除指定事件指定回调，或移除整个事件
  // ...
};
```

`_events` 是实例上的对象，key 是事件名，value 是回调数组。`$emit` 遍历数组依次执行。

## 弊端

**1. 内存泄漏（最常见坑）**

组件销毁时如果忘记 `$off`，回调仍挂在 bus 上，下次 `$emit` 会触发已销毁组件的回调，导致报错或内存泄漏。

**2. 全局命名空间冲突**

所有事件共享一个 bus，事件名容易重复（A 组件的 `refresh` 和 B 组件的 `refresh` 冲突）。

**3. 数据流不可追溯**

`$emit` 是"广播"，任何组件都能触发，调试时难以定位"是谁触发的"、"谁在监听"。相比 Vuex 的"状态变更可追踪"，EventBus 是隐式的。

**4. 无类型支持**

`$bus.$on('refresh', (data) => ...)` 的 `data` 是 any，TS 无法推断。

**5. 与响应式理念冲突**

Vue 推崇"数据驱动视图"，EventBus 是"命令式通信"，容易导致状态散落在事件回调里，难以维护。

## Vue3 的变化

**Vue3 移除了 `$on/$off/$once`**，只保留 `$emit`（用于父子组件通信）。原因：

1. 官方不推荐用实例做事件总线。
2. 事件总线模式有上述诸多弊端。
3. 推荐用更合适的方案替代。

**Vue3 的替代方案：**

```js
// 1. mitt / tiny-emitter（第三方库）
import mitt from 'mitt';
const bus = mitt();
bus.on('refresh', handler);
bus.emit('refresh', data);
bus.off('refresh', handler);

// 2. 响应式状态共享（推荐）
import { reactive } from 'vue';
const state = reactive({ count: 0 });
export { state }; // 多组件共享同一响应式对象

// 3. Pinia（大型应用）
import { defineStore } from 'pinia';
export const useStore = defineStore('main', () => {
  const count = ref(0);
  return { count };
});
```
