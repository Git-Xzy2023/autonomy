# 什么是 DNS？

DNS（域名系统）是互联网上作为域名和 IP 地址相互映射的一个分布式数据库，使用户更方便地访问互联网，而不用记住能够被机器直接读取的 IP 数串。

## DNS 的查询方式

| 方式 | 说明 |
|------|------|
| **递归查询** | 客户机向本地 DNS 服务器发请求，如果该 DNS 服务器不知道结果，则以 DNS 客户的身份继续向其它 DNS 服务器查询 |
| **迭代查询** | DNS 服务器之间查询时，不是直接替它询问，而是返回参考结果，让它自己再去问 |

## 为什么域名解析用 UDP？

因为 UDP 快，TCP 慢。UDP 报文头部只有 8 字节（TCP 20 字节），UDP 不需要先建立连接和维持连接，也没有超时重传，速度比 TCP 快很多。当 DNS 报文大小超过 512 字节时会启用 TCP。

## 如何优化 DNS 查询

1. **DNS 缓存**：浏览器缓存已解析的域名 IP 地址
2. **DNS 预解析**：`<link rel="dns-prefetch" href="https://example.com">` 提前解析域名
3. **合理配置 DNS 服务器**：选择稳定的公共 DNS（如 8.8.8.8、114.114.114.114）
4. **减少 DNS 请求次数**：减少页面中不同域名的数量
