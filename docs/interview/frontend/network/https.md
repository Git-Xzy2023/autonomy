# HTTP 和 HTTPS 的区别

## 基本概念

**HTTP**：超文本传输协议，客户端和服务器端请求和应答的标准（TCP），用于从 WWW 服务器传输超文本到本地浏览器。

**HTTPS**：HTTP 的安全版，HTTP 协议加上 SSL/TLS 协议构成的可进行加密传输、身份认证的网络协议。HTTP 端口 80，HTTPS 端口 443。

## HTTP 的不足之处

- **通信使用明文**，内容可能被窃听
- **不验证通信方身份**，可能遭遇伪装
- **无法证明报文完整性**，可能已遭篡改

## 区别对比

| 方面 | HTTP | HTTPS |
|------|------|-------|
| 协议 | 明文传输 | SSL 加密传输 |
| 连接方式 | 无状态 | SSL+HTTP，比 HTTP 安全 |
| 端口 | 80 | 443 |
| 证书 | 不需要 | 需要 CA 申请证书 |
| 资源消耗 | 较低 | 页面加载延长约 50%，增加 10%-20% 耗电 |
| 安全性 | 数据未加密，不安全 | 防止数据被窃取、改变 |
