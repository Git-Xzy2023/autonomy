# 什么是 XSS 攻击？

## 概念

XSS（跨站脚本攻击）是一种代码注入攻击，攻击者通过在网站注入恶意脚本，使之在用户的浏览器上运行，从而盗取用户的信息如 Cookie 等。

XSS 的本质是因为网站没有对恶意代码进行过滤，与正常代码混合在一起，浏览器无法分辨哪些脚本是可信的。

**攻击者可进行以下操作**：

- 获取页面数据（DOM、Cookie、localStorage）
- DOS 攻击，占用服务器资源
- 破坏页面结构
- 流量劫持

## 攻击类型

| 类型 | 说明 |
|------|------|
| **存储型** | 恶意脚本存储在目标服务器上（如论坛发帖、评论），用户请求时从服务器返回并执行 |
| **反射型** | 恶意代码放在 URL 中，服务器将恶意代码拼接在 HTML 中返回执行 |
| **DOM 型** | 前端 JavaScript 从 URL 中取出恶意代码并执行，属于前端自身安全漏洞 |

## 防御 XSS

- 对插入到 HTML 中的代码做好充分的转义
- 使用 CSP（Content Security Policy），通过 HTTP 头或 meta 标签开启白名单
- Cookie 使用 http-only，使脚本无法获取
- 使用验证码，避免脚本伪装成用户执行操作
