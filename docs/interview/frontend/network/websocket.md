# WebSocket

## 对 WebSocket 的理解

WebSocket 是 HTML5 提供的浏览器与服务器进行全双工通讯的网络技术，属于应用层协议。基于 TCP，复用 HTTP 的握手通道，浏览器和服务器只需要一次握手即可创建持久性连接。

**特点**：
- 支持双向通信，实时性强
- 可发送文本和二进制数据
- 建立在 TCP 之上，服务端实现较容易
- 数据格式轻量，性能开销小
- 没有同源限制
- 协议标识符为 ws/wss，默认端口 80/443
- 与 HTTP 兼容性好

## 即时通讯实现方式对比

| 方式 | 原理 | 优缺点 |
|------|------|--------|
| **短轮询** | 浏览器每隔一段时间发送 HTTP 请求 | 实现简单，但频繁请求浪费资源 |
| **长轮询** | 服务器挂起请求，有数据才返回 | 减少请求次数，但连接挂起也浪费资源 |
| **SSE** | 服务端使用流信息向客户端推送 | 单向通信，基于 HTTP，兼容性好 |
| **WebSocket** | 全双工协议，双向推送 | 性能最好，服务端配置较复杂 |

**性能**：WebSocket > SSE > 长轮询 > 短轮询
**兼容性**：短轮询 > 长轮询 > SSE > WebSocket
