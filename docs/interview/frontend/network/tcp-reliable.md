# TCP 的可靠传输机制

TCP 的可靠传输基于**连续 ARQ 协议**和**滑动窗口协议**。

发送方维持一个**发送窗口**，窗口内包含已发送但未确认的报文段和允许发送但还未发送的报文段。

**超时重传**：定时器超时，重新发送所有已发送但未收到确认的报文段，并将超时间隔设为以前的两倍。

**快速重传**：发送方收到接收方的三个冗余确认应答后，在当前定时器结束前，发送所有已发送但未确认的报文段。

**累计确认**：接收方按序到达才返回肯定回答，乱序则丢弃并返回最近按序到达的确认号。累计确认保证了返回的确认号之前的报文段都已按序到达。

**发送窗口大小**由接收窗口剩余大小和网络中拥塞程度共同决定，TCP 通过控制发送窗口的长度来控制报文段的发送速率。
