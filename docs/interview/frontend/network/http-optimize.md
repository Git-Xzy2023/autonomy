# HTTP 如何优化？

基于"让通信线路更快"这个前提，优化方案如下：

1. **CDN 托管**：把静态资源放在离用户更近的地方，减少请求往返时间
2. **缓存处理**：协商缓存、强制缓存等，减少重复请求
3. **请求合并**：减少 HTTP 请求次数（雪碧图、CSS/JS 压缩、图片合并、Base64 内嵌图片）
4. **GZIP 压缩**：减小请求/响应的大小
5. **开启 HTTPS**：对数据传输安全有帮助
6. **支持 HTTP2.0**：多路复用、头部压缩、服务端推送
7. **使用合理域名**：静态资源使用独立域名，减少请求时附带不必要的 Cookie
8. **DNS 预解析**：减少 DNS 解析时间
9. **使用 CDN**：让用户从最近的节点获取资源
10. **合理使用缓存**：合理设置缓存策略
11. **减少请求数**：合并 CSS、JS 文件
