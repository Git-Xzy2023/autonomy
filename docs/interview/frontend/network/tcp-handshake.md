# TCP 的三次握手和四次挥手

## 三次握手

三次握手是指建立一个 TCP 连接时，客户端和服务器总共发送 3 个包。主要作用：确认双方的接收能力和发送能力是否正常、指定初始化序列号。

1. **第一次握手**：客户端给服务端发 SYN 报文，指明客户端的初始化序列号 ISN，客户端进入 SYN_SEND 状态
2. **第二次握手**：服务器收到后，以自己的 SYN 报文作为应答，指定自己的 ISN，同时把客户端的 ISN + 1 作为 ACK 值，服务器进入 SYN_REVD 状态
3. **第三次握手**：客户端收到后，发送 ACK 报文，把服务器的 ISN + 1 作为 ACK 值，客户端进入 ESTABLISHED 状态

**为什么要三次握手？** 防止失效的连接请求报文段被服务器接收。如果只用两次握手，服务端无法知道自己的序号是否已被确认，也无法处理网络中滞留的失效连接请求。

## 四次挥手

1. **第一次挥手**：客户端发送 FIN 报文，停止发送数据，进入 FIN_WAIT1 状态
2. **第二次挥手**：服务端收到 FIN 后，发送 ACK 报文，进入 CLOSE_WAIT 状态。客户端收到后进入 FIN_WAIT2 状态
3. **第三次挥手**：服务端所有数据发送完毕后，发送 FIN 报文，进入 LAST_ACK 状态
4. **第四次挥手**：客户端收到 FIN 后，发送 ACK 报文，进入 TIME_WAIT 状态，经过 2MSL 后进入 CLOSED 状态

**为什么需要四次挥手？** TCP 是全双工的，服务端收到 FIN 后可能还有数据要发送，不能像三次握手一样 SYN+ACK 一起发送。

**为什么客户端要等待 2MSL？** 防止发送给服务器的确认报文段丢失，确保服务端能正常关闭。
