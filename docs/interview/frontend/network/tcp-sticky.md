# TCP 粘包问题

## 为什么会粘包？

TCP 连接默认启用 **Nagle 算法**，短时间多个数据会缓冲到一起作一次发送，减少 IO 消耗提高性能。连续两次 send 发送 data1 和 data2，接收端可能出现：

- 先收到 data1 的部分数据 + data2 的全部
- 先收到 data1 的全部 + data2 的部分数据
- 一次性收到 data1 和 data2 的全部数据

这就是粘包。

## 解决方案

| 方案 | 说明 | 适用场景 |
|------|------|----------|
| **间隔等待** | 等一段时间再进行下一次 send | 交互频率低 |
| **关闭 Nagle 算法** | `socket.setNoDelay()` 让每次 send 都不缓冲 | 数据量大且频率不高 |
| **封包/拆包** | 给每个数据包前后加特征数据，收到时根据特征分割 | 业内常见方案 |

## 为什么 UDP 不会粘包？

UDP 是面向消息的协议，每个 UDP 包有消息头（来源地址、端口等），以消息为单位提取数据。UDP 具有保护消息边界，接收端一次只能接收一个数据包，天然不会粘包。
