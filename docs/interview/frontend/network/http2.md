# HTTP 与 HTTP2.0 的区别

## HTTP/1.x 的缺点

1. **队头阻塞**：多个请求共用一个 TCP 连接，同一时刻只能处理一个请求
2. **多路复用开销大**：使用多个 TCP 连接（6-8 个）来解决，但带来握手成本和慢启动问题
3. **HTTP 头部太大**：重复的请求头增加开销（User-Agent、Cookie 等）
4. **不支持服务器推送**：必须由客户端请求后服务器才能返回数据

## HTTP2.0 的优势

1. **多路复用**：多个请求可同时在一个 TCP 连接中并发传输，可独立设置优先级
2. **二进制分帧**：将信息分为更小的消息和帧，采用二进制格式编码
3. **头部压缩**：使用 HPACK 算法对请求头和响应头进行压缩
4. **服务器推送**：服务端可主动向客户端推送资源
5. **更安全**：通常使用 HTTPS 加密传输
6. **流**：连接中的虚拟信道，每个流有唯一的整数标识符

## HTTP2.0 多路复用的原理

HTTP/2 采用二进制帧，每个请求/响应数据被拆分成多个帧（头部帧携带头部信息，数据帧携带 body 数据）。通过多路复用，多个请求和响应可在同一个 TCP 连接中同时传输，而 HTTP/1.x 需要为每个域名建立 6-8 个 TCP 连接来实现并发。
