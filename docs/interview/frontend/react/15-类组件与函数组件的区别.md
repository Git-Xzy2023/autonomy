# 类组件与函数组件的区别

**Q1: 类组件和函数组件有什么区别？**

| 特性             | 类组件                              | 函数组件                          |
| ---------------- | ----------------------------------- | --------------------------------- |
| 定义方式         | `class Comp extends React.Component` | `function Comp(props) {}`         |
| 状态             | `this.state` + `this.setState`      | `useState`                        |
| 生命周期         | 生命周期方法（componentDidMount 等）| `useEffect` 等 Hooks              |
| this             | 有 this，需要绑定事件处理器         | 无 this                           |
| 实例             | 每个组件创建一个实例                | 无实例，纯函数调用                |
| 性能             | 略重（创建实例）                    | 略轻（函数调用）                  |
| 逻辑复用         | HOC、Render Props                   | 自定义 Hook（更优雅）             |
| Ref 转发         | `forwardRef`                        | `forwardRef` + `useImperativeHandle` |
| 错误边界         | ✅ 可实现（getDerivedStateFromError）| ❌ 无法实现（目前）               |

**Q2: 函数组件相比类组件有哪些优势？**

1. **更简洁**：没有 `this`、`bind`、构造函数等样板代码。

2. **更易理解**：函数组件是纯函数，输入 props 输出 UI，没有实例和生命周期的复杂概念。

3. **逻辑复用更优雅**：自定义 Hook 解决了类组件中 HOC 嵌套地狱和 Render Props 的性能问题。

```jsx
// 类组件：逻辑复用靠 HOC，容易嵌套地狱
const EnhancedComp = withAuth(withTheme(withRouter(MyComp)));

// 函数组件：自定义 Hook，扁平清晰
function MyComp() {
  useAuth();
  const theme = useTheme();
  const router = useRouter();
  return <div>...</div>;
}
```

4. **更利于 Tree Shaking**：未使用的 Hooks 可以被打包工具移除。

**Q3: 类组件的 this 有哪些坑？**

```jsx
class Button extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
    // 方式 1：在构造函数中 bind
    this.handleClick = this.handleClick.bind(this);
  }

  // 方式 2：类字段语法（推荐）
  handleClick = () => {
    this.setState({ count: this.state.count + 1 });
  };

  // 如果不绑定，handleClick 作为 onClick 回调时 this 会丢失
  // handleClick() {
  //   this.setState(...) // ❌ this is undefined
  // }

  render() {
    return (
      <button onClick={this.handleClick}>
        {/* 方式 3：箭头函数（每次渲染创建新函数，可能影响子组件性能） */}
        <button onClick={(e) => this.handleClick(e)}>
          {this.state.count}
        </button>
      </button>
    );
  }
}
```

**Q4: 函数组件能完全替代类组件吗？**

在 Hooks 出现后，函数组件能覆盖 99% 的场景。但有一个例外：**错误边界**目前只能用类组件实现。

```jsx
// 错误边界必须是类组件
class ErrorBoundary extends React.Component {
  state = { hasError: false };

  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  componentDidCatch(error, info) {
    logError(error, info);
  }

  render() {
    if (this.state.hasError) {
      return <h1>Something went wrong.</h1>;
    }
    return this.props.children;
  }
}
```

React 官方也提到未来可能为函数组件增加错误边界支持，但目前（2026年）仍需类组件。

**Q5: 函数组件每次渲染都会执行，那性能会不会更差？**

函数组件每次渲染都会重新执行整个函数，但：

1. React 内部做了大量优化（Fiber、调度、Diff）
2. 可以用 `React.memo` 包裹组件避免不必要的重渲染
3. 函数组件没有实例创建的开销
4. 实际项目中，函数组件的性能通常**优于或等于**类组件
