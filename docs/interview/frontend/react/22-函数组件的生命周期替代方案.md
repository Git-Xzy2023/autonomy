# 函数组件的生命周期替代方案

**Q1: 函数组件没有生命周期，如何替代类组件的生命周期？**

函数组件通过 Hooks 来模拟类组件的生命周期：

| 类组件生命周期 | 函数组件替代方案 |
| --- | --- |
| `constructor` | `useState` 初始化 |
| `componentDidMount` | `useEffect(() => {}, [])` |
| `componentDidUpdate` | `useEffect(() => {}, [deps])` |
| `componentWillUnmount` | `useEffect` 的 cleanup 函数 |
| `shouldComponentUpdate` | `React.memo` + `useMemo`/`useCallback` |
| `getDerivedStateFromProps` | 渲染期间直接计算（不存 state） |
| `getSnapshotBeforeUpdate` | `useLayoutEffect` |
| `componentDidCatch` | 暂无替代（仍需类组件错误边界） |

**Q2: 如何用 useEffect 模拟 componentDidMount？**

```jsx
function App() {
  // 空依赖数组：只在挂载后执行一次
  useEffect(() => {
    console.log('组件挂载');

    // 模拟 componentWillUnmount
    return () => {
      console.log('组件卸载');
    };
  }, []); // ← 空数组

  return <div>Hello</div>;
}
```

**Q3: 如何用 useEffect 模拟 componentDidUpdate？**

```jsx
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  // 依赖 userId：userId 变化时重新执行
  useEffect(() => {
    fetch(`/api/users/${userId}`)
      .then(res => res.json())
      .then(setUser);
  }, [userId]); // ← 依赖数组

  // ⚠️ 注意：不加依赖数组会每次渲染都执行
  // useEffect(() => { ... }); // 相当于每次 render 都执行

  return <div>{user?.name}</div>;
}
```

**Q4: useEffect 和类组件生命周期有什么本质区别？**

1. **执行时机不同**：`componentDidMount/Update` 在 DOM 更新后**同步**执行；`useEffect` 在浏览器绘制后**异步**执行（不阻塞屏幕更新）。

2. **cleanup 执行顺序不同**：

```jsx
// 类组件：先执行上一个 props 的 componentWillUnmount，再执行新的 componentDidMount
// 函数组件：先执行新的 effect，再执行上一个 effect 的 cleanup

function Timer({ delay }) {
  useEffect(() => {
    const id = setInterval(() => console.log('tick'), delay);
    return () => {
      console.log('清理上一个定时器');
      clearInterval(id);
    };
  }, [delay]);

  // 当 delay 从 1000 变成 2000 时：
  // 1. 执行新的 effect（设置 2000ms 的定时器）
  // 2. 执行上一个 cleanup（清理 1000ms 的定时器）
}
```

3. **闭包特性**：每次渲染 useEffect 都会捕获当次渲染的 props 和 state。

```jsx
function Counter() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    const timer = setInterval(() => {
      // 每次渲染的 effect 都捕获了当次的 count
      // 如果不更新，这里永远是 0
      setCount(count + 1);
    }, 1000);
    return () => clearInterval(timer);
  }, [count]); // ← count 变化会重新设置定时器

  // ✅ 更好的方案：函数式更新，不依赖 count
  // useEffect(() => {
  //   const timer = setInterval(() => {
  //     setCount(c => c + 1);
  //   }, 1000);
  //   return () => clearInterval(timer);
  // }, []); // ← 只设置一次

  return <div>{count}</div>;
}
```

**Q5: 如何模拟 shouldComponentUpdate？**

```jsx
// 类组件
class MyComp extends React.Component {
  shouldComponentUpdate(nextProps) {
    return this.props.value !== nextProps.value;
  }
  render() {
    return <div>{this.props.value}</div>;
  }
}

// 函数组件：用 React.memo
const MyComp = React.memo(({ value }) => {
  return <div>{value}</div>;
});

// 自定义比较函数（默认是浅比较）
const MyComp = React.memo(({ value, data }) => {
  return <div>{value}</div>;
}, (prevProps, nextProps) => {
  // 返回 true 表示"相等"不更新，false 表示"不等"要更新
  return prevProps.value === nextProps.value &&
         prevProps.data.id === nextProps.data.id;
});
```

**Q6: useLayoutEffect 和 useEffect 在生命周期中的位置？**

```
React 更新 DOM
  ↓
useLayoutEffect 执行（同步，在浏览器绘制前）
  ↓
浏览器绘制屏幕
  ↓
useEffect 执行（异步，在绘制后）
```

`useLayoutEffect` 适合需要**在用户看到画面之前**读取 DOM 布局并同步修改的场景（如测量元素尺寸后调整样式）。
