# 类组件的生命周期

**Q1: 类组件的生命周期分为哪几个阶段？**

React 类组件的生命周期分为三个阶段：**挂载**、**更新**、**卸载**。

```
挂载阶段（Mounting）：
  constructor
    ↓
  getDerivedStateFromProps（静态方法）
    ↓
  render
    ↓
  componentDidMount

更新阶段（Updating）：
  getDerivedStateFromProps
    ↓
  shouldComponentUpdate
    ↓
  render
    ↓
  getSnapshotBeforeUpdate
    ↓
  componentDidUpdate

卸载阶段（Unmounting）：
  componentWillUnmount

错误处理：
  getDerivedStateFromError（render 阶段）
  componentDidCatch（commit 阶段）
```

**Q2: 各个生命周期方法的作用？**

```jsx
class MyComponent extends React.Component {
  constructor(props) {
    super(props);
    // 1. 初始化 state
    // 2. 绑定事件处理函数
    // 3. 不要在这里调用 setState
    this.state = { count: 0 };
  }

  // 罕见用法：当 props 变化时同步到 state
  // 返回对象来更新 state，返回 null 不更新
  static getDerivedStateFromProps(props, state) {
    if (props.initialCount !== state.prevCount) {
      return {
        count: props.initialCount,
        prevCount: props.initialCount,
      };
    }
    return null;
  }

  // 决定是否重渲染，返回 false 则跳过 render
  // 用于性能优化
  shouldComponentUpdate(nextProps, nextState) {
    return this.props.value !== nextProps.value;
  }

  // 渲染 UI（纯函数，不能有副作用）
  render() {
    return <div>{this.state.count}</div>;
  }

  // 组件挂载后调用（DOM 已生成）
  // 适合：发请求、添加事件监听、设置定时器、操作 DOM
  componentDidMount() {
    this.fetchData();
    window.addEventListener('resize', this.handleResize);
  }

  // 在 DOM 更新前获取信息（如滚动位置）
  // 返回值会传给 componentDidUpdate
  getSnapshotBeforeUpdate(prevProps, prevState) {
    return this.listRef.current.scrollHeight;
  }

  // 更新完成后调用
  // 适合：DOM 操作、网络请求（需比较 prevProps）
  componentDidUpdate(prevProps, prevState, snapshot) {
    if (this.props.id !== prevProps.id) {
      this.fetchData();
    }
  }

  // 组件卸载前调用
  // 适合：清除定时器、移除事件监听、取消请求
  componentWillUnmount() {
    window.removeEventListener('resize', this.handleResize);
    clearInterval(this.timer);
  }
}
```

**Q3: 哪些生命周期被废弃了？为什么？**

React 16.3 废弃了以下三个生命周期（17 版本后如果存在会报警告，18 版本后如果开启严格模式会报错）：

| 废弃的方法 | 替代方案 |
| --- | --- |
| `componentWillMount` | `constructor` 或 `componentDidMount` |
| `componentWillReceiveProps` | `getDerivedStateFromProps` |
| `componentWillUpdate` | `getSnapshotBeforeUpdate` + `componentDidUpdate` |

废弃原因：这些方法在 Fiber 架构的**异步渲染**下可能被多次调用，导致副作用重复执行，引发 bug。

**Q4: componentDidMount 和 componentDidUpdate 中发请求有什么区别？**

```jsx
class UserList extends React.Component {
  state = { users: [] };

  componentDidMount() {
    // 首次挂载时加载
    this.fetchUsers(this.props.page);
  }

  componentDidUpdate(prevProps) {
    // props 变化时重新加载
    if (this.props.page !== prevProps.page) {
      this.fetchUsers(this.props.page);
    }
  }

  fetchUsers = (page) => {
    fetch(`/api/users?page=${page}`)
      .then(res => res.json())
      .then(users => this.setState({ users }));
  };

  render() {
    return <ul>{this.state.users.map(u => <li key={u.id}>{u.name}</li>)}</ul>;
  }
}
```

**Q5: getSnapshotBeforeUpdate 的使用场景？**

在 DOM 更新前捕获一些信息（如滚动位置），更新后恢复。

```jsx
class ChatList extends React.Component {
  // 场景：聊天列表新增消息时，保持滚动位置不跳动
  getSnapshotBeforeUpdate(prevProps) {
    if (prevProps.messages.length < this.props.messages.length) {
      const list = this.listRef.current;
      return list.scrollHeight - list.scrollTop; // 记录距离底部
    }
    return null;
  }

  componentDidUpdate(prevProps, prevState, snapshot) {
    if (snapshot !== null) {
      const list = this.listRef.current;
      list.scrollTop = list.scrollHeight - snapshot; // 恢复滚动位置
    }
  }

  render() {
    return <div ref={this.listRef} style={{ height: 300, overflow: 'auto' }}>
      {this.props.messages.map(m => <div key={m.id}>{m.text}</div>)}
    </div>;
  }
}
```
