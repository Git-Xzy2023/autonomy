# 代码分割与懒加载

**Q1: 什么是代码分割？为什么需要它？**

代码分割是将打包后的 JS 文件拆分成多个小文件，**按需加载**，减少首屏加载时间。

```
不分割：
  bundle.js (2MB) → 首屏加载 2MB

分割后：
  main.js (200KB) → 首屏加载
  vendor.js (500KB) → 首屏加载
  AdminPage.js (300KB) → 访问时加载
  Dashboard.js (200KB) → 访问时加载
```

**Q2: React.lazy 如何实现路由级懒加载？**

```jsx
import { lazy, Suspense } from 'react';

// 懒加载组件
const Home = lazy(() => import('./pages/Home'));
const About = lazy(() => import('./pages/About'));
const Dashboard = lazy(() => import('./pages/Dashboard'));

function App() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/dashboard" element={<Dashboard />} />
      </Routes>
    </Suspense>
  );
}
// 访问 / 时只加载 Home.js
// 访问 /about 时才加载 About.js
```

**Q3: Suspense 的 fallback 是什么？**

在懒加载组件加载完成前显示的占位 UI。

```jsx
<Suspense fallback={<Spinner />}>
  <LazyComponent />
</Suspense>

// 可以嵌套
<Suspense fallback={<PageSkeleton />}>
  <Header />
  <Suspense fallback={<ContentSkeleton />}>
    <LazyContent />
  </Suspense>
  <Footer />
</Suspense>
// Header 和 Footer 立即显示
// LazyContent 加载时显示 ContentSkeleton
```

**Q4: 如何实现组件级懒加载？**

```jsx
// 场景：重量级组件（图表编辑器、富文本编辑器）按需加载
const ChartEditor = lazy(() => import('./ChartEditor'));

function App() {
  const [showEditor, setShowEditor] = useState(false);

  return (
    <div>
      <button onClick={() => setShowEditor(true)}>打开编辑器</button>
      {showEditor && (
        <Suspense fallback={<div>加载编辑器...</div>}>
          <ChartEditor />
        </Suspense>
      )}
    </div>
  );
}
```

**Q5: Webpack 的魔法注释是什么？**

```jsx
// 给分割的 chunk 命名
const Home = lazy(() => import(
  /* webpackChunkName: "home" */
  './pages/Home'
));

// 预加载：在浏览器空闲时加载
const About = lazy(() => import(
  /* webpackChunkName: "about" */
  /* webpackPrefetch: true */
  './pages/About'
));
// <link rel="prefetch" href="about.js"> 会在空闲时加载

// 预获取：和当前路由一起加载（优先级更高）
const Dashboard = lazy(() => import(
  /* webpackChunkName: "dashboard" */
  /* webpackPreload: true */
  './pages/Dashboard'
));
// <link rel="preload" href="dashboard.js">
```

**prefetch vs preload：**

| 特性       | prefetch              | preload               |
| ---------- | --------------------- | --------------------- |
| 加载时机   | 浏览器空闲时          | 与当前页面并行        |
| 优先级     | 低                    | 高                    |
| 使用场景   | 未来可能访问的页面    | 当前页面即将需要的资源|

**Q6: 如何做第三方库的代码分割？**

```jsx
// 1. 动态 import
async function loadLodash() {
  const _ = await import('lodash');
  return _.default;
}

// 2. SplitChunksPlugin（webpack.config.js）
module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
        },
      },
    },
  },
};

// 3. 按需引入减少体积
// ❌ 引入整个 lodash
import _ from 'lodash';

// ✅ 按需引入
import debounce from 'lodash/debounce';
```

**Q7: React.lazy 有什么限制？**

1. 只能用于默认导出的组件
2. 不能用于服务端渲染（SSR 用 `@loadable/component`）
3. 必须配合 Suspense 使用

```jsx
// ❌ 命名导出不能用 React.lazy
export const MyComponent = () => <div>Hi</div>;
const Lazy = lazy(() => import('./MyComponent')); // 报错

// ✅ 改为默认导出
export default function MyComponent() {
  return <div>Hi</div>;
}

// ✅ 或者中间层转换
const MyComponent = lazy(() =>
  import('./MyComponent').then(mod => ({ default: mod.MyComponent }))
);
```
