# 路由守卫的实现

**Q1: 什么是路由守卫？**

路由守卫是在路由跳转前进行**权限验证**的机制，决定是否允许访问目标路由。

常见场景：
- 未登录用户不能访问需要认证的页面
- 普通用户不能访问管理员页面
- 未完成的表单离开前提示确认

**Q2: React Router v6 如何实现路由守卫？**

React Router 没有内置守卫 API，通过**封装组件**实现。

```jsx
// 1. 认证守卫
function RequireAuth({ children }) {
  const { user } = useAuth();
  const location = useLocation();

  if (!user) {
    // 未登录：重定向到登录页，并记住来源
    return <Navigate to="/login" state={{ from: location }} replace />;
  }

  return children;
}

// 2. 角色守卫
function RequireRole({ role, children }) {
  const { user } = useAuth();

  if (user?.role !== role) {
    return <Navigate to="/403" replace />;
  }

  return children;
}

// 使用
<Routes>
  <Route path="/login" element={<Login />} />
  <Route path="/" element={<RequireAuth><Layout /></RequireAuth>}>
    <Route index element={<Home />} />
    <Route path="profile" element={<Profile />} />
    <Route
      path="admin"
      element={
        <RequireRole role="admin">
          <Admin />
        </RequireRole>
      }
    />
  </Route>
</Routes>
```

**Q3: 如何实现登录后跳回原页面？**

```jsx
// RequireAuth 中记录来源
function RequireAuth({ children }) {
  const { user } = useAuth();
  const location = useLocation();

  if (!user) {
    return <Navigate to="/login" state={{ from: location }} replace />;
  }
  return children;
}

// Login 组件中跳回
function Login() {
  const navigate = useNavigate();
  const location = useLocation();
  const { login } = useAuth();

  const from = location.state?.from?.pathname || '/';

  const handleLogin = async () => {
    await login(username, password);
    navigate(from, { replace: true }); // 跳回原页面
  };

  return <button onClick={handleLogin}>登录</button>;
}
```

**Q4: 如何实现离开页面确认？**

```jsx
// 使用 useBlocker（React Router v6.4+）
function UnsavedForm() {
  const [isDirty, setIsDirty] = useState(false);
  const blocker = useBlocker(
    ({ currentLocation, nextLocation }) =>
      isDirty && currentLocation.pathname !== nextLocation.pathname
  );

  return (
    <>
      <form onChange={() => setIsDirty(true)}>
        {/* 表单内容 */}
      </form>

      {blocker.state === 'blocked' && (
        <ConfirmDialog
          message="您有未保存的更改，确定离开吗？"
          onConfirm={() => blocker.proceed()}
          onCancel={() => blocker.reset()}
        />
      )}
    </>
  );
}

// React Router v6.4 之前用 Prompt 组件
// <Prompt when={isDirty} message="您有未保存的更改，确定离开吗？" />
```

**Q5: 如何实现全局路由 loading？**

```jsx
// 数据路由（v6.4+ loader）
const router = createBrowserRouter([
  {
    path: '/users/:id',
    loader: async ({ params }) => {
      const res = await fetch(`/api/users/${params.id}`);
      return res.json();
    },
    element: <UserDetail />,
  },
]);

function UserDetail() {
  const user = useLoaderData(); // 直接获取 loader 加载的数据
  return <div>{user.name}</div>;
}

// 全局 loading
function App() {
  const navigation = useNavigation();
  return (
    <div>
      {navigation.state === 'loading' && <GlobalSpinner />}
      <RouterProvider router={router} />
    </div>
  );
}
```

**Q6: 如何实现基于权限的动态路由？**

```jsx
function App() {
  const { user } = useAuth();
  const routes = generateRoutes(user);

  return (
    <Routes>
      {routes.map(route => (
        <Route key={route.path} {...route} />
      ))}
    </Routes>
  );
}

// 根据用户权限生成路由
function generateRoutes(user) {
  const allRoutes = [
    { path: '/', element: <Home />, public: true },
    { path: '/profile', element: <Profile />, auth: true },
    { path: '/admin', element: <Admin />, auth: true, role: 'admin' },
    { path: '/editor', element: <Editor />, auth: true, role: 'editor' },
  ];

  return allRoutes.filter(route => {
    // 公开路由所有人可访问
    if (route.public) return true;
    // 需要登录的路由检查用户
    if (route.auth && !user) return false;
    // 需要特定角色
    if (route.role && user?.role !== route.role) return false;
    return true;
  });
}
```

**Q7: 路由守卫和 HTTP 拦截器的区别？**

| 特性       | 路由守卫                    | HTTP 拦截器                |
| ---------- | --------------------------- | -------------------------- |
| 作用层面   | 前端路由                    | 网络请求                   |
| 检查时机   | 路由跳转前                  | 请求发送前/响应返回后      |
| 主要目的   | 页面访问权限                | 请求鉴权、错误处理         |
| 两者关系   | 互补，都需要                | 互补，都需要               |

```jsx
// 路由守卫：防止用户看到不该看的页面
<RequireAuth>
  <Dashboard />
</RequireAuth>

// HTTP 拦截器：防止未授权的 API 请求
axios.interceptors.request.use(config => {
  const token = localStorage.getItem('token');
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

axios.interceptors.response.use(
  res => res,
  error => {
    if (error.response?.status === 401) {
      // token 过期，跳转登录
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);
```
