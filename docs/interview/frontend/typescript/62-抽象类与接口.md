# 抽象类与接口

**Q50: 抽象类（abstract class）和接口（interface）有什么区别？**

```ts
// 抽象类 —— 可以有实现代码
abstract class Animal {
  abstract makeSound(): void; // 抽象方法：子类必须实现
  move(): void {
    // 普通方法：有默认实现
    console.log("Moving...");
  }
}

// 接口 —— 只有声明，没有实现
interface Flyable {
  fly(): void;
}

class Bird extends Animal implements Flyable {
  makeSound() {
    console.log("Chirp");
  }
  fly() {
    console.log("Flying");
  }
}
```

| 特性                | 抽象类                         | 接口                       |
| ------------------- | ------------------------------ | -------------------------- |
| 有实现代码          | ✅                             | ❌                         |
| 有构造函数          | ✅                             | ❌                         |
| 有成员字段/状态     | ✅                             | ❌                         |
| 多继承              | ❌（一个类只能继承一个抽象类） | ✅（`implements A, B, C`） |
| 编译到 JS           | ✅（会生成 class 代码）        | ❌（完全消失）             |
| 可使用 `instanceof` | ✅                             | ❌                         |
