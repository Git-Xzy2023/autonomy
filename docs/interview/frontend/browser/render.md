# 浏览器的渲染过程

## 渲染步骤

1. 解析收到的文档，构建 **DOM 树**
2. 解析 CSS，生成 **CSSOM 规则树**
3. 根据 DOM 树和 CSSOM 树构建**渲染树**
4. 根据渲染树进行**布局（回流）**
5. **绘制**阶段，将内容显示在屏幕上

注意：这个过程是逐步完成的，渲染引擎会尽可能早地将内容呈现到屏幕上，不会等到所有 HTML 解析完成之后再构建和布局渲染树。

## 浏览器渲染优化

**针对 JavaScript：**
- 尽量将 JS 文件放在 body 的最后
- body 中间尽量不要写 `<script>` 标签
- 使用 async 和 defer 异步引入

script / async / defer 区别：
| 方式 | 行为 |
|------|------|
| **script** | 立即停止渲染去加载资源，加载后立即执行 |
| **async** | 异步加载，加载后立即执行，不能保证加载顺序 |
| **defer** | 异步加载，等 DOM 树解析完再执行，按顺序执行 |

**针对 CSS：**
- 使用 link 而非 @import
- CSS 少时采用内嵌样式

**减少回流与重绘：**
- 操作 DOM 时尽量在低层级节点操作
- 不使用 table 布局
- 使用 absolute/fixed 使元素脱离文档流
- 避免频繁操作 DOM，使用文档片段 documentFragment
