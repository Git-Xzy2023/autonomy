# 浏览器事件机制

## 事件模型

现代浏览器共有三种事件模型：
1. **DOM0 级事件模型**：不会传播，没有事件流概念
2. **IE 事件模型**：事件处理阶段 + 事件冒泡阶段
3. **DOM2 级事件模型**：事件捕获阶段 + 事件处理阶段 + 事件冒泡阶段

## 阻止事件冒泡

```javascript
event.stopPropagation(); // 标准浏览器
event.cancelBubble = true; // IE 浏览器
```

## 事件委托

利用事件冒泡机制，把子节点的监听函数定义在父节点上，由父节点统一处理多个子元素的事件。

**特点**：减少内存消耗、动态绑定事件

**局限性**：focus、blur 等没有冒泡机制的无法使用事件委托。

## 事件触发过程

1. **window 到目标节点**：事件捕获阶段
2. **目标节点**：事件处理阶段
3. **目标节点到 window**：事件冒泡阶段
