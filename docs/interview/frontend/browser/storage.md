# 浏览器存储方式

## localStorage

- 生命周期永久，除非主动删除
- 存储大小约 5MB
- 同源策略限制
- 仅在客户端保存，不参与服务端通信

## sessionStorage

- 临时保存同一窗口（或标签页）的数据
- 刷新页面不删除，关闭窗口/标签页后删除
- 同源策略限制，且只在同一浏览器的同一窗口下共享

## Cookie 字段

| 字段 | 说明 |
|------|------|
| Name | Cookie 名称 |
| Value | Cookie 的值 |
| Domain | 可访问该 Cookie 的域名 |
| Path | 可访问此 Cookie 的页面路径 |
| Expires/Max-Age | 超时时间 |
| Secure | 是否使用 HTTPS 发送 |
| HTTPOnly | 能否通过脚本访问 |

## 三者区别

| 特性 | Cookie | localStorage | sessionStorage |
|------|--------|--------------|----------------|
| 容量 | 4KB | 5MB | 5MB |
| 生命周期 | 由 expires 指定 | 永久保存 | 关闭窗口后删除 |
| 同源限制 | 同源可访问 | 同源可访问 | 同源同窗口可访问 |
| 与服务端通信 | 每次请求自动携带 | 不参与通信 | 不参与通信 |

## IndexedDB 特点

- **键值对储存**：所有类型数据可直接存入
- **异步**：操作时不锁死浏览器
- **支持事务**：一步失败，整个事务取消
- **同源限制**：只能访问自身域名下的数据库
- **储存空间大**：一般不少于 250MB
- **支持二进制储存**：可储存 ArrayBuffer 和 Blob 对象
