# 什么是 CSRF 攻击？

## 概念

CSRF（跨站请求伪造）：攻击者诱导用户进入一个第三方网站，该网站向被攻击网站发送跨站请求。利用用户在被攻击网站的登录状态，冒充用户执行操作。

**本质**：利用 Cookie 在同源请求中携带发送给服务器的特点，实现用户冒充。

## 攻击类型

| 类型 | 方式 |
|------|------|
| GET 类型 | 在 img 标签中构建请求，用户打开页面自动发起 |
| POST 类型 | 构建隐藏表单，用户进入页面自动提交 |
| 链接类型 | 在 a 标签 href 中构建请求，诱导用户点击 |

## 防御方式

- **同源检测**：验证请求头中的 Origin 或 Referer 字段
- **CSRF Token**：服务器返回随机数 Token，请求时验证
- **Cookie 双重验证**：在 Cookie 和请求参数中放入随机字符串，服务器比对
- **设置 Samesite**：限制 Cookie 不能被第三方使用
