# 浏览器缓存机制

## 缓存全过程

1. 浏览器第一次加载资源，服务器返回 200，浏览器下载并缓存资源文件与 response header
2. 下一次加载时，先检查**强缓存**，比较当前时间与上次返回 200 的时间差，没超过 max-age 则命中强缓存，直接从本地读取
3. 强缓存过期，进入**协商缓存**，向服务器发送带 If-None-Match 和 If-Modified-Since 的请求
4. 服务器优先根据 Etag 判断文件是否修改：
   - Etag 一致 → 返回 304，使用本地缓存
   - 不一致 → 返回新资源和新 Etag，返回 200

## 强缓存 vs 协商缓存

**强缓存**设置方式：
- **Expires**（HTTP/1.0）：指定资源过期绝对时间
- **Cache-Control**（HTTP/1.1）：优先级高于 Expires

Cache-Control 字段：
| 字段 | 说明 |
|------|------|
| `public` | 可被任何对象缓存 |
| `private` | 只能被用户浏览器缓存 |
| `no-cache` | 无强缓存，有协商缓存 |
| `no-store` | 禁止任何缓存 |
| `max-age=xxx` | 缓存最大有效期（秒） |
| `s-maxage=xxx` | 优先级高于 max-age，仅用于共享缓存（CDN） |

**协商缓存**设置方式：
- **Last-Modified / If-Modified-Since**：资源最后修改时间（精确到秒）
- **Etag / If-None-Match**：资源唯一标识符，优先级更高

## 为什么需要浏览器缓存？

- 减少服务器负担，提高网站性能
- 加快客户端网页加载速度
- 减少多余网络数据传输
