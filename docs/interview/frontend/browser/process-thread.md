# 进程与线程的概念

## 基本概念

- **进程**：CPU 资源分配的最小单位，描述了 CPU 在运行指令及加载和保存上下文所需的时间，放在应用上说代表了一个程序
- **线程**：CPU 调度的最小单位，是进程中的更小单位，描述了执行一段指令所需的时间

## 进程和线程的关系

1. 进程中的任意一线程执行出错，都会导致整个进程的崩溃
2. 线程之间共享进程中的数据
3. 当一个进程关闭后，操作系统会回收进程所占用的内存
4. 进程之间的内容相互隔离

## Chrome 浏览器架构

最新的 Chrome 浏览器包含：

- **1 个浏览器主进程**：界面显示、用户交互、子进程管理、存储
- **1 个 GPU 进程**：实现 3D CSS 效果、绘制 Chrome UI
- **1 个网络进程**：页面网络资源加载
- **多个渲染进程**：核心任务是将 HTML/CSS/JS 转换为网页，Blink 和 V8 运行在此
- **多个插件进程**：运行插件，隔离崩溃影响

打开一个网页最少需要 **4 个进程**：网络进程 + 浏览器进程 + GPU 进程 + 渲染进程。

## 进程和线程的区别

| 特性 | 进程 | 线程 |
|------|------|------|
| 定义 | 独立应用 | 不能单独存在 |
| 资源 | CPU 资源分配的最小单位 | CPU 调度的最小单位 |
| 通信 | 需借助进程间通信 | 可直接共享进程资源 |
| 调度 | 切换开销大 | 切换开销小 |
| 系统开销 | 创建/撤销开销大 | 创建/撤销开销小 |
