# 数据库学习指南

> 数据库是后端开发的核心基础设施。本章节涵盖关系型数据库 MySQL、键值缓存 Redis、文档型数据库 MongoDB，从基础语法到索引原理、事务、性能优化。

---

## 一、为什么学习数据库

- 💾 **数据持久化**：程序重启数据不丢失
- ⚡ **高性能查询**：索引、缓存加速访问
- 🔒 **事务保证**：ACID 特性保证数据一致性
- 📈 **面试高频**：索引原理、事务隔离、SQL 优化是后端必考
- 🏗️ **架构核心**：缓存、主从、分库分表是高并发基础

## 三、数据库分类

```
数据库
├── 关系型（RDBMS）
│   ├── MySQL（最流行，互联网首选）
│   ├── PostgreSQL（功能丰富，复杂查询）
│   └── SQLite（轻量嵌入式）
│
├── 非关系型（NoSQL）
│   ├── 键值型：Redis（缓存之王）
│   ├── 文档型：MongoDB（灵活 schema）
│   ├── 列族型：HBase（海量数据）
│   └── 图形型：Neo4j（关系网络）
│
└── 搜索引擎
    └── Elasticsearch（全文搜索）
```

---

## 四、推荐学习路线

1. **MySQL 基础**：SQL 语法、表设计、索引
2. **MySQL 进阶**：事务、锁、执行计划、优化
3. **Redis**：数据结构、持久化、缓存模式
4. **MongoDB**：文档模型、聚合管道
