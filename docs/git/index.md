# Git 版本控制学习指南

> Git 是目前最流行的分布式版本控制系统，是团队协作开发的基石。本章节系统整理 Git 的核心概念、常用命令、分支策略、工作流与最佳实践。

---

## 一、为什么学习 Git

- 🔄 **版本管理**：记录文件每一次修改，随时回退
- 🤝 **团队协作**：多人并行开发，分支互不干扰
- 🛡️ **安全可靠**：分布式架构，本地完整仓库
- 🚀 **效率提升**：分支轻量，切换迅速
- 📦 **生态完善**：GitHub / GitLab / Gitee 等平台

## 三、核心概念总览

```
工作区 (Working Directory)
   ↓ git add
暂存区 (Staging Area / Index)
   ↓ git commit
本地仓库 (Local Repository)
   ↓ git push
远程仓库 (Remote Repository)
```

---

## 四、推荐学习路线

1. **基础**：理解三个区域，掌握 add/commit/push/pull
2. **分支**：学会 branch/merge/rebase，理解分支模型
3. **工作流**：选择适合团队的分支策略
4. **进阶**：hooks、submodule、bisect、reflog 等高级技巧
