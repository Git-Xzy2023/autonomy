# 小程序开发学习指南

本章节涵盖小程序开发的完整流程，按照 **创建小程序 → 项目结构与配置 → 开发详解 → 部署与发布 → 跨端开发** 的路线系统化整理，以微信小程序为主线，兼顾跨端方案。

---

## 一、为什么学习小程序？

小程序是一种不需要下载安装即可使用的应用，用户扫一扫或搜一下即可打开，实现了"触手可及"的梦想。

- 🚀 **无需下载**：用完即走，降低用户使用门槛
- 📱 **体验接近原生**：流畅的交互和丰富的设备能力
- 🔗 **微信生态**：10+ 亿微信用户，便捷传播
- 💰 **开发成本低**：一套代码多端运行，相比原生 App 成本大幅降低
- 🏢 **企业必备**：电商、服务、工具类应用的标配入口

---

## 二、技术栈总览

| 技术             | 定位              | 适用场景                 |
| ---------------- | ----------------- | ------------------------ |
| **微信小程序**   | 原生开发          | 微信生态内的应用         |
| **支付宝小程序** | 原生开发          | 支付宝生态内的应用       |
| **uni-app**      | 跨端框架（Vue）   | 一套代码发布多端         |
| **Taro**         | 跨端框架（React） | 一套代码发布多端         |
| **WXML/WXSS**    | 模板与样式语言    | 小程序原生开发基础       |
| **WXS**          | 视图层脚本        | 模板内数据处理，性能优化 |

---

## 三、推荐学习路线

```
创建小程序 → 项目结构与配置 → 开发详解 → 部署与发布
   ↓
注册账号     目录结构        WXML 语法    上传代码
安装工具     app.json        数据绑定     体验版测试
创建项目     页面配置        事件处理     提交审核
版本体系     分包配置        组件开发     发布上线
域名配置     npm 支持        常用 API     CI/CD 自动化
   ↓
跨端开发（按需学习）
├── uni-app（Vue 生态）
└── Taro（React 生态）
```

## 五、跨端框架对比速查表

| 特性           | 微信原生       | uni-app        | Taro             |
| -------------- | -------------- | -------------- | ---------------- |
| **开发框架**   | 原生 WXML/JS   | Vue            | React / Vue      |
| **支持平台**   | 仅微信         | 全平台         | 全平台           |
| **学习成本**   | 中             | 低（Vue 用户） | 中               |
| **性能**       | 最佳           | 良好           | 良好             |
| **生态**       | 微信官方       | 插件市场丰富   | React 生态       |
| **TypeScript** | 需配置         | ✅             | ✅               |
| **App 支持**   | ❌             | ✅（nvue）     | ✅（RN）         |
| **H5 支持**    | ❌             | ✅             | ✅               |
| **适合场景**   | 仅微信、高性能 | 多端、Vue 团队 | 多端、React 团队 |

## 六、学习建议

1. **新手入门**：从 [创建小程序](/web/miniprogram/01-create/) 开始，完成账号注册和项目创建。
2. **掌握基础**：学习 [项目结构与配置](/web/miniprogram/02-structure/) 和 [开发详解](/web/miniprogram/03-development/)，理解小程序的开发模式。
3. **实际上线**：跟随 [部署与发布](/web/miniprogram/04-deploy/) 完成第一个小程序的上线流程。
4. **跨端扩展**：如果需要支持多端，根据团队技术栈选择 [uni-app](/web/miniprogram/05-cross-platform/) 或 Taro。

接下来，从你感兴趣的模块开始学习吧！
