# 部署与发布

> 本章讲解小程序从开发完成到上线的完整流程，包括代码上传、体验版测试、提交审核、发布上线、版本管理和灰度发布。

---

## 一、发布流程总览

```
┌─────────────────────────────────────────────────────────────┐
│                     小程序发布流程                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  开发完成 ──▶ 上传代码 ──▶ 体验版测试 ──▶ 提交审核          │
│                 │             │              │              │
│                 ▼             ▼              ▼              │
│              开发版        体验版         审核中            │
│                                                             │
│  ─────────────────────────────────────────────────────      │
│                                                             │
│  审核通过 ──▶ 发布上线 ──▶ 灰度发布 ──▶ 全量发布            │
│                 │             │              │              │
│                 ▼             ▼              ▼              │
│              待发布        部分用户        所有用户          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.1 版本流转

| 阶段       | 版本状态   | 可见人群             | 操作位置         |
| ---------- | ---------- | -------------------- | ---------------- |
| 开发调试   | 开发版     | 开发者本人           | 开发者工具       |
| 内部测试   | 体验版     | 体验成员（≤15人）    | 微信公众平台     |
| 等待审核   | 审核中     | 不可见               | 微信公众平台     |
| 审核完成   | 待发布     | 不可见               | 微信公众平台     |
| 线上运行   | 已发布     | 所有微信用户         | 微信公众平台     |

## 二、上传代码

### 2.1 上传步骤

1. **在开发者工具中点击「上传」按钮**

2. **填写版本信息**：

```
版本号：1.0.0（遵循语义化版本）
项目备注：首次发布，包含首页、个人中心、订单功能
```

3. **点击确定，等待上传完成**

4. **上传成功后，登录微信公众平台查看**

### 2.2 上传前检查清单

上传代码前，请逐项检查：

- [ ] **AppID 正确**：确保使用的是正式 AppID，非测试号
- [ ] **服务器域名已配置**：所有接口域名已在后台配置
- [ ] **关闭 debug 模式**：`app.json` 中 `"debug": false`
- [ ] **移除 console.log**：生产环境不应有调试日志
- [ ] **检查环境变量**：API 地址切换为生产环境
- [ ] **代码压缩**：开发者工具自动压缩
- [ ] **包大小检查**：主包 ≤ 2MB，总包 ≤ 20MB
- [ ] **无未使用的代码和资源**
- [ ] **图标和图片已优化**
- [ ] **测试通过**：功能、兼容性、性能测试

### 2.3 包大小优化

如果上传时提示包大小超限，可采取以下优化措施：

**1. 使用分包**

```json
{
  "pages": ["pages/index/index"],
  "subPackages": [
    {
      "root": "packageA",
      "pages": ["pages/order/list"]
    }
  ]
}
```

**2. 图片优化**

- 使用网络图片替代本地图片
- 压缩图片（推荐使用 [TinyPNG](https://tinypng.com/)）
- 使用 iconfont 或字体图标替代图片
- 删除未使用的图片资源

**3. 代码优化**

- 删除未使用的 npm 包
- 使用按需引入（如 ECharts 按需加载）
- 开启 `lazyCodeLoading: "requiredComponents"`

**4. 分包预下载**

```json
{
  "preloadRule": {
    "pages/index/index": {
      "network": "wifi",
      "packages": ["packageA"]
    }
  }
}
```

### 2.4 查看包大小

在开发者工具中：

- 菜单「详情」→「基本信息」可查看代码包大小
- 或使用 `wx.getAccountInfoSync()` 获取信息

## 三、体验版测试

### 3.1 设置体验版

1. 登录 [微信公众平台](https://mp.weixin.qq.com)
2. 进入「管理」→「版本管理」
3. 在「开发版本」列表中找到刚上传的版本
4. 点击「选为体验版」

### 3.2 添加体验成员

1. 进入「管理」→「成员管理」
2. 在「体验成员」区域点击「+」
3. 输入微信号，发送邀请
4. 被邀请人微信确认后成为体验成员

> 💡 体验成员最多 15 人，包括开发者和管理员。

### 3.3 体验版二维码

- 体验版设置成功后，会生成专属二维码
- 体验成员用微信扫码即可打开体验版
- 也可在微信「小程序」列表中找到体验版入口

### 3.4 体验版测试要点

| 测试项         | 检查内容                                   |
| -------------- | ------------------------------------------ |
| **功能测试**   | 所有功能流程是否正常                       |
| **网络请求**   | 接口是否正常返回（生产环境域名）           |
| **真机兼容**   | iOS、Android 不同机型测试                  |
| **性能测试**   | 启动速度、页面切换流畅度                   |
| **弱网测试**   | 在弱网环境下是否有合理提示                 |
| **权限测试**   | 授权弹窗是否正常、拒绝授权后处理是否合理   |
| **支付测试**   | 支付流程是否正常（需配置支付能力）         |
| **分享测试**   | 分享给好友、朋友圈是否正常                 |

### 3.5 真机调试

除了扫码体验，还可以使用真机调试：

1. 开发者工具点击「真机调试」
2. 手机微信扫码连接
3. 可以在电脑上查看真机的 Console、Network 等

> 💡 真机调试比体验版更适合排查问题，可以看到日志。

## 四、提交审核

### 4.1 审核前准备

**1. 完善小程序信息**

- 小程序名称、图标、简介
- 服务类目（必须与实际功能匹配）
- 隐私协议（如收集用户信息）

**2. 配置服务器域名**

- 确保所有接口域名已配置
- 域名必须 HTTPS、已备案

**3. 检查合规性**

- 内容合法合规
- 无诱导分享、诱导下载
- 无虚拟支付（iOS）
- 符合[小程序运营规范](https://developers.weixin.qq.com/miniprogram/product/)

### 4.2 提交审核步骤

1. 登录微信公众平台
2. 进入「管理」→「版本管理」
3. 在「体验版」或「开发版本」中点击「提交审核」
4. 填写审核信息：

```
版本描述：本次更新内容说明
测试账号：（如需登录，提供测试账号密码）
功能页面：列出核心功能页面路径
```

5. 提交后状态变为「审核中」

### 4.3 审核时间

| 审核类型     | 预计时间     | 说明                       |
| ------------ | ------------ | -------------------------- |
| 首次审核     | 1-7 个工作日 | 新发布的小程序             |
| 常规更新     | 1-3 个工作日 | 已上线小程序的版本更新     |
| 加急审核     | 数小时       | 部分账号可申请加急         |

### 4.4 审核结果

- **通过**：状态变为「待发布」
- **不通过**：查看驳回原因，修改后重新提交

### 4.5 常见审核驳回原因

| 原因                       | 解决方案                           |
| -------------------------- | ---------------------------------- |
| **服务类目不符**           | 修改类目或调整功能                 |
| **功能不完整**             | 完善功能，确保核心流程可用         |
| **存在诱导行为**           | 移除诱导分享、关注等内容           |
| **内容违规**               | 删除违规内容                       |
| **未配置域名**             | 配置服务器域名                     |
| **测试账号无法登录**       | 提供正确的测试账号                 |
| **iOS 虚拟支付**           | 移除 iOS 端的虚拟支付功能          |
| **隐私协议缺失**           | 添加隐私协议页面                   |
| **获取权限未说明用途**     | 在申请权限时说明用途               |
| **页面空白或报错**         | 排查生产环境问题                   |

## 五、发布上线

### 5.1 全量发布

审核通过后：

1. 进入「版本管理」
2. 在「审核版本」中点击「发布」
3. 选择发布方式：
   - **全量发布**：立即对所有用户生效
   - **分阶段发布**：灰度发布，逐步扩大范围

### 5.2 灰度发布（分阶段发布）

灰度发布可以逐步将新版本推送给部分用户，降低风险。

```
┌────────────────────────────────────────┐
│            灰度发布比例                 │
├────────────────────────────────────────┤
│                                        │
│  10% ──▶ 30% ──▶ 50% ──▶ 100%         │
│                                        │
│  每个阶段观察 1-2 天，无问题再扩大     │
│                                        │
└────────────────────────────────────────┘
```

**操作步骤**：

1. 点击「发布」
2. 选择「分阶段发布」
3. 设置初始比例（如 10%）
4. 观察数据，逐步提高比例
5. 最终 100% 全量发布

**灰度发布适用场景**：

- 大版本更新
- 核心功能重构
- 数据库结构变更
- 风险较高的改动

### 5.3 撤销发布

如果发布后发现严重问题：

1. 进入「版本管理」
2. 在「线上版本」中点击「回退」
3. 回退到上一个稳定版本

> ⚠️ **注意**：回退功能有次数限制，且只能回退到上一个版本。建议发布前充分测试。

### 5.4 版本回退注意事项

- 回退后用户需要重新打开小程序才能生效
- 回退不影响已缓存的用户数据
- 回退后仍可重新发布新版本

## 八、小程序更新机制

### 8.1 自动更新

小程序有自动更新机制，但用户可能不会立即获取新版本：

```
用户打开小程序
     │
     ▼
检查是否有新版本
     │
     ├── 有 ──▶ 下载新版本 ──▶ 下次冷启动使用新版本
     │
     └── 无 ──▶ 使用当前缓存版本
```

### 8.2 强制更新

如果需要用户立即更新，可以使用 `UpdateManager`：

```javascript
// app.js
App({
  onLaunch() {
    this.checkUpdate()
  },

  checkUpdate() {
    const updateManager = wx.getUpdateManager()

    // 检查是否有新版本
    updateManager.onCheckForUpdate((res) => {
      console.log('是否有新版本', res.hasUpdate)
    })

    // 新版本下载完成
    updateManager.onUpdateReady(() => {
      wx.showModal({
        title: '更新提示',
        content: '新版本已就绪，是否重启应用？',
        success(res) {
          if (res.confirm) {
            // 应用新版本并重启
            updateManager.applyUpdate()
          }
        }
      })
    })

    // 新版本下载失败
    updateManager.onUpdateFailed(() => {
      wx.showModal({
        title: '更新失败',
        content: '新版本下载失败，请检查网络后重试',
        showCancel: false
      })
    })
  }
})
```

- 进入「推广」→「搜索」→「自定义关键词」

## 十、CI/CD 自动化部署

### 10.1 miniprogram-ci

微信官方提供了 [miniprogram-ci](https://developers.weixin.qq.com/miniprogram/dev/devtools/ci.html) 工具，支持命令行上传代码。

**安装**：

```bash
npm install miniprogram-ci -D
```

**使用**：

```javascript
// upload.js
const ci = require('miniprogram-ci')

const project = new ci.Project({
  appid: 'wx1234567890abcdef',
  type: 'miniProgram',
  projectPath: './',
  privateKeyPath: './private.key', // 从后台下载的上传密钥
  ignores: ['node_modules/**/*']
})

async function upload() {
  const result = await ci.upload({
    project,
    version: '1.2.0',
    desc: 'CI 自动上传',
    setting: {
      es6: true,
      minify: true,
      autoPrefixWXSS: true
    },
    onProgressUpdate: console.log
  })
  console.log('上传成功', result)
}

upload().catch(console.error)
```

### 10.2 GitHub Actions 示例

```yaml
# .github/workflows/deploy.yml
name: Deploy MiniProgram

on:
  push:
    tags:
      - 'v*'

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - uses: actions/setup-node@v3
        with:
          node-version: '18'

      - name: Install dependencies
        run: npm ci

      - name: Upload to WeChat
        env:
          PRIVATE_KEY: ${{ secrets.WX_PRIVATE_KEY }}
        run: |
          echo "$PRIVATE_KEY" > private.key
          node upload.js
```

### 10.3 获取上传密钥

1. 登录微信公众平台
2. 进入「开发」→「开发管理」→「开发设置」
3. 在「小程序代码上传密钥」中下载密钥文件
4. 将密钥文件加入 `.gitignore`，不要提交到仓库

## 十一、总结

### ✅ 关键知识点

1. **发布流程**：上传代码 → 体验版测试 → 提交审核 → 发布上线
2. **上传代码**：填写版本号和备注，注意包大小限制
3. **体验版测试**：添加体验成员，全面测试后再提交审核
4. **审核要点**：合规性、功能完整性、域名配置、隐私协议
5. **灰度发布**：逐步扩大发布范围，降低风险
6. **版本管理**：规范版本号，填写清晰备注
7. **更新机制**：使用 UpdateManager 实现强制更新
8. **CI/CD**：使用 miniprogram-ci 实现自动化上传
